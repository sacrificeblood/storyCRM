# Доска связок — сервер (упрощённая версия, без логина)

Свой Node.js/Express-сервер + база данных Postgres. Экран входа убран —
сайт открывается сразу, данные общие для всех, у кого есть ссылка на сайт.
Это проще и надёжнее для внутреннего командного инструмента, но помните:
защиты паролем больше нет — не публикуйте ссылку широко.

## Из чего состоит

```
server.js      — Express: REST-хранилище для доски + /api/health для проверки базы
db.js          — подключение к Postgres и создание таблиц при первом запуске
public/index.html — сама доска (фронтенд), стартует сразу, без экрана входа
.env.example   — какие переменные окружения нужны
```

## Проверка базы одной ссылкой

Откройте `https://ваш-сайт.onrender.com/api/health` в браузере. Если всё в порядке, увидите:
```json
{"ok":true,"db":"connected","serverTime":"...","savedKeys":1}
```
Если база не подключена — увидите `{"ok":false,"db":"error","error":"..."}` с текстом ошибки.
Это самый быстрый способ понять, жива ли база, без всей остальной доски.

## Шаг 1. Бесплатная база Postgres

Если уже создавали (Neon/Supabase) — можно использовать ту же самую, ничего пересоздавать не нужно.

## Шаг 2. Настроить .env

```
DATABASE_URL=<строка подключения>
NODE_ENV=production
PORT=3000
```

## Шаг 3. Запуск локально

```
npm install
npm start
```
Откройте http://localhost:3000 — доска должна открыться сразу, без входа.

## Шаг 4. Деплой на Render (или другой бесплатный Node-хостинг)

Как раньше: залить папку на GitHub → Render → New Web Service → Build command `npm install`,
Start command `npm start` → в Environment добавить `DATABASE_URL` и `NODE_ENV=production`.

## Важно про безопасность

Без логина — любой человек, у которого есть ссылка на сайт, может читать и менять данные.
Для внутреннего инструмента команды это обычно приемлемо (ссылку просто не публикуют),
но если понадобится закрыть доступ — можно вернуть простую защиту паролем без полноценной
регистрации (один общий пароль на команду, без email-аккаунтов) — скажите, добавлю.
